本页面汇总了 **CURC 2027 ROBOCON 赛季** 新人入队、留校培训、集中培训、考核与自学的全部信息。请仔细阅读并按步骤完成。

## 集中培训安排

- **时间**：六月底至七月初
- **覆盖组别**：所有组别（机械 / 硬件 / 软件 / 视觉 / 运营 / 项管 / 财务合规）
- **负责人**：领队 **@朱佩韦**

集中培训期间会统一讲解赛季方向、技术栈、组内分工与基础工具链，是新人快速融入团队的最佳时机。

## 自学建议

## 第一次考核：学习进度更新

请大家把**正在学习什么**或**正在做什么**，更新到对应组别目录下。提交一次算作：

1. **本次招新工作的第一次考核**
2. **Gitee 操作的考核任务**

> 学习进度本身也会作为招新考核内容的一部分。

### 提交位置

- 仓库分支：`feat/curc2027-onboarding`
- 目录：`competitions/CURC2027ROBOCON`
- 链接：[https://gitee.com/darrenpig/new_energy_coder_club/tree/feat/curc2027-onboarding/competitions/CURC2027ROBOCON](https://gitee.com/darrenpig/new_energy_coder_club/tree/feat/curc2027-onboarding/competitions/CURC2027ROBOCON)

### 提交步骤

<Steps>
  <Step title="Fork 或拉取分支">
    切换到 `feat/curc2027-onboarding` 分支，或在 Gitee 上基于此分支创建自己的工作分支。
  </Step>
  <Step title="在对应组别目录更新">
    找到自己所属组别目录，新增或更新学习进度文件（Markdown / 文本均可），说明正在学习的技术、遇到的问题、阶段性成果。
  </Step>
  <Step title="提交并推送">
    按团队版 Gitee 提交教程完成 commit 与 push。建议 commit message 格式：`docs(curc2027): 组别-姓名-学习进度更新`。
  </Step>
  <Step title="发起 PR">
    向 `feat/curc2027-onboarding` 分支发起 Pull Request，等待审核。
  </Step>
</Steps>

## 考核说明

- 考核**不做刻意规定要求**，会根据实际时机随缘安排。
- 除学习进度更新外，也可以通过对团队的实际贡献参与考核。
- 群里会不定期发布团队承接的**小项目**，欢迎新人认领。
