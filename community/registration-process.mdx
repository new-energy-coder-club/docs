## 活动背景

开源之夏是中国科学院软件研究所发起的"点亮计划"系列活动，2026 年正式步入 **2.0 时代**——以基础软件为技术核心，以 **RISC-V 为生态重心**。活动联合各大开源组织，针对重要开源软件的开发与维护提供项目开发任务，并向全球高校学生开放报名。

2026 年采用 **1+1 双模式并行**：**经典模式** + **全新领航模式（Pioneer Mode）**。

> 历史数据：累计 **288** 家开源社区、**3487** 个项目任务、来自 **1026** 所高校的 **12257** 名学生报名，成功解决 **2794** 个实际项目问题。

## 一、谁能报名？（社区/企业资格）

### 🏢 企业（满足任一条件）

1. 拥有 **RISC-V 软硬件产品**
2. 企业为 **RVI（RISC-V International）会员**
3. 有开源软件上游面向 **RV 架构支持的贡献记录**

### 👥 开源组织

- 以 **核心基础软件类** 为主，具体范围：
  - 操作系统、系统固件、编译器与工具链、系统基础库、语言运行时、虚拟化基础、调试与性能分析、AI 核心框架、安全基础
- 仓库采用 **OSI 认证开源协议** 或 OKF 推荐开放协议
- 公开运营 **半年以上**

## 二、经典模式 — 社区报名流程

```
注册/登录 → 提交组织资料 → 组委会审核 → 上线官网 → 发布项目 → 导师验证 → 项目审核上线
```

### Step 1：组织报名（全年持续开放）

| 事项 | 说明 |
|---|---|
| 入口 | 活动官网 [summer.ospp.ac.cn](https://summer.ospp.ac.cn/) 组织登录入口 |
| 新组织 | 注册组织账号，填写提交组织资料 |
| 往届组织 | 用原账号登录，**更新资料后重新提交审核** |
| 审核 | 组委会审核通过即报名成功，组织陆续上线官网 |

> ⚠️ 组织联络人与导师身份重合时，请使用**不同邮箱**注册系统。

### Step 2：提交项目（全年持续开放）

**组织联络人**通过组织账号在系统内提交项目，每个项目需包含：

| 字段 | 要求 |
|---|---|
| 项目名称 | 明确体现技术+目标任务 |
| 项目描述 | 背景、已有工作、不足、改进要点、最终目标 |
| 项目难度 | 基础级 / 进阶级 |
| 技术领域 & 编程语言 | — |
| 项目产出要求 | 可量化，明确学生需要做什么、预期效果 |
| 项目技术要求 | 编程语言、技术栈、开发经验等 |
| 项目成果仓库 | 已有仓库地址（非新建空仓） |
| 开发周期 | **1-3 个月**（以月为单位） |
| 预估工时 | 参考值 |
| 学习资料 | 参考书籍、论文、博客、issue 链接等 |

**项目类型要求**：
- 组委会全额支持：**核心基础软件 + RISC-V 生态**相关项目
- 尚未支持 RISC-V 的组织，应发布一个 RISC-V 类项目并完成结项
- 非上述范围：组织可**自费支持**（承担结项报酬），需签合作协议

### Step 3：指定导师 + 导师身份验证

1. 组织账号为每个项目指定一名导师（系统内添加导师信息）
2. 系统自动向导师发邀请邮件 → 导师点链接登录完成身份验证
3. **未完成身份验证的项目无法审核发布**
4. 一位导师最多指导 **2 个项目**
5. 🚨 导师不得同时以学生身份参加活动

### Step 4：组委会项目审核 → 上线官网

审核通过的项目陆续上线官网展示。

### Step 5：学生申请审核（持续进行）

- 导师应在收到学生申请后 **一周内审核**
- 无人申请/无人中选的项目持续开放
- 导师审核通过 → 组委会审核 → 中选确认
- **申请书质量是最主要标准**
- ❌ 不允许提前内定学生、私下承诺中选或要求学生提前开发

### Step 6：项目开发 + 结项

- 学生按开发周期完成开发，提交结项报告 + PR/MR
- 导师结项审核 → 组委会成果审核 → 发放报酬

**经典模式报酬**：基础级 3500 元 / 进阶级 5000 元（税前）

## 三、领航模式 — 社区报名流程

> ⚠️ **首期限定为组委会邀请制**，受邀社区参与。

```
受邀 → 发布需求方向(1-3个) → 开放Pitch时段 → 学生投递提案 → 社区组织Pitch选拔 →
生成项目任务 → 提交系统 → 学生提交正式申请书 → 开发 → 结项
```

### Step 1：发布需求方向

受邀社区在系统内完成组织报名后，发布 **1-3 个需求方向**，每个方向填写：

| 字段 | 要求 |
|---|---|
| **方向名称** | 简洁明确，如"RISC-V 驱动性能优化" |
| **方向简述** | 2-3 句话说明关注什么（不描述具体任务） |
| **方向重要性** | 对社区的价值，解决什么痛点或技术债 |
| **对学生的期望** | 需要哪些背景知识或能力 |
| **社区提供支持** | 如：每周一次 30min 线上同步、开发板远程访问、文档/测试集、代码导读 |
| **建议切入点**（可选） | 社区已想到但没精力做的具体方向 |
| **提案接收渠道** | 社区指定邮箱或其他渠道（**系统外提交**） |
| **Pitch 时段** | 如："2026 年 X 月 X 日 19:00-21:00" 或 "另行通知" |

### Step 2：学生提交简易提案

学生选择感兴趣的方向 → 深入社区挖掘待解决问题 → 撰写简易提案 → 投递至社区指定渠道。

### Step 3：社区开放 Pitch，选拔提案

- 社区定期集中组织提案沟通（线上会议或其他形式）
- 即时确认提案是否通过
- 选定学生和提案

### Step 4：生成正式项目任务

- 导师+学生共同细化需求和产出
- 基于选定提案生成具体项目任务（格式同经典模式）
- 由社区账号提交至系统并指定学生账号
- 学生提交正式申请书

### Step 5：开发 + 结项

- 开发周期：**2-6 个月**（以月为单位）
- 导师结项审核 → 组委会成果审核 → 发放报酬

**领航模式报酬**：基础级 6000 元 / 进阶级 8000 元 / 挑战级 10000 元（税前）

## 四、两种模式对比

| 维度 | 经典模式 | 领航模式 |
|---|---|---|
| **参与方式** | 公开报名 | 组委会邀请制（首期） |
| **主导方** | 社区发布具体项目 | 社区给方向，**学生定义问题** |
| **项目来源** | 社区/导师预定义 | 学生提案 → 师生共同细化 |
| **提案渠道** | 活动系统内 | 社区指定外部渠道（邮箱等） |
| **选拔方式** | 导师审核申请书 | Pitch 沟通 + 即时确认 |
| **开发周期** | 1-3 个月 | 2-6 个月 |
| **报酬档位** | 3500 / 5000 | 6000 / 8000 / 10000 |
| **难度档位** | 基础、进阶 | 基础、进阶、**挑战** |

## 五、附录：后台管理系统表单字段详解

> 来源：组织指南 + 后台管理系统 portal.summer.ospp.ac.cn

### A.1 组织报名表单

#### 基本信息区

| 字段 | 必填 | 说明 |
|---|---|---|
| **组织名称** | ✅ | 组织全称，需与营业执照/注册文件一致。往届组织用原账号登录后需更新资料重新提交 |
| **组织简称** | ✅ | 简洁缩写或常用名，用于官网列表展示 |
| **组织类型** | ✅ | 下拉选择：开源组织 / 企业（企业需满足 RISC-V 相关条件） |
| **组织 Logo** | 否 | 建议上传正方形清晰 Logo，用于官网展示 |
| **组织官网** | ✅ | 以 `https://` 开头的完整网址 |
| **组织代码仓库地址** | ✅ | 以 `https://` 开头的仓库地址（GitHub / Gitee 等） |
| **组织所在国家/地区** | ✅ | 下拉选择，默认"中国大陆" |

#### 描述信息区

| 字段 | 必填 | 说明 |
|---|---|---|
| **组织简介** | ✅ | 简要介绍组织背景、核心方向、社区规模等（官网列表展示用） |
| **组织详细描述** | ✅ | 详细介绍：技术领域、已有成果、社区治理模式、参与开源之夏的意愿和目标 |
| **技术标签** | 否 | 从系统标签库中选择，如"操作系统""编译器""RISC-V""AI"等，帮助学生在官网筛选 |

#### 联系人信息区

| 字段 | 必填 | 说明 |
|---|---|---|
| **联系人姓名** | ✅ | 组织联络人真实姓名，作为组委会与组织的沟通桥梁 |
| **联系人邮箱** | ✅ | 接收系统通知和审核结果。⚠️ 若联络人同时担任导师，需用**不同邮箱**注册导师账号 |
| **联系人手机号** | 否 | 备用联系方式 |

#### 资质信息区

| 字段 | 必填 | 说明 |
|---|---|---|
| **组织注册文件/营业执照** | 否 | 企业需上传营业执照；开源组织可上传社区注册/备案文件，用于资质审核 |

#### 提交操作

| 按钮 | 说明 |
|---|---|
| **提交审核** | 确认信息无误后提交，组委会审核通过即上线官网 |
| **保存草稿** | 未完成时暂存，不影响审核状态 |
| **重置** | 清空当前表单所有内容 |
| **返回** | 返回上一页，注意先保存 |

### A.2 经典模式 — 项目发布字段

完成组织报名后，在系统内逐个项目发布：

| 字段 | 必填 | 说明 |
|---|---|---|
| **项目名称** | ✅ | 明确体现技术方向+目标任务。示例："RISC-V 平台上的 OpenSSL 性能优化" |
| **项目描述** | ✅ | 包含：相关背景、已有工作基础、存在不足、改进要点、最终实现目标 |
| **项目难度** | ✅ | 下拉选择：**基础级**（3500元）/ **进阶级**（5000元） |
| **技术领域** | ✅ | 如：操作系统、编译器、数据库、AI 框架等 |
| **编程语言** | ✅ | 项目主要使用的编程语言 |
| **项目产出要求** | ✅ | **可量化**描述学生需完成什么、达到什么效果。如"实现 XX 模块，通过 YY 基准测试，性能提升 ZZ%" |
| **项目技术要求** | ✅ | 对学生的技能要求，如"熟悉 C 语言、了解 RISC-V 指令集、有 Linux 内核开发经验" |
| **项目成果仓库** | ✅ | 已有项目仓库地址（**不能是新建空仓**） |
| **项目开发周期** | ✅ | **1-3 个月**（以月为单位） |
| **预估工时** | ✅ | 参考值，如"每周约 20 小时" |
| **项目备注/学习资料** | 否 | 推荐预习的参考书籍、论文、博客、相关 issue 链接等 |

> ⚠️ **类型约束**：组委会全额支持「核心基础软件+RISC-V 生态」项目；非此类项目需组织**自费**承担报酬并签合作协议。

### A.3 领航模式 — 需求方向发布字段

首期仅限受邀社区。每个社区发布 **1-3 个方向**：

| 字段 | 必填 | 说明 |
|---|---|---|
| **方向名称** | ✅ | 简洁明确。示例："RISC-V 驱动性能优化""基础库的跨平台适配""固件构建系统改进" |
| **方向简述** | ✅ | 2-3 句话说明关注什么，**不描述具体任务**。示例："我们关注 RISC-V 平台上的驱动性能优化，尤其是外设驱动的中断延迟、DMA 传输效率以及多核环境下的并发访问开销。" |
| **方向重要性** | ✅ | 对社区的价值，解决什么用户痛点或技术债。示例："随着 RISC-V 硬件走向高性能场景，驱动性能已成用户反馈主要瓶颈，直接影响生态吸引力。" |
| **对学生的期望** | ✅ | 学生需要具备哪些背景知识或能力 |
| **社区提供支持** | ✅ | 如：每周一次 30min 线上同步、开发板远程访问、性能分析文档/测试集、相关模块代码导读 |
| **建议切入点** | 否 | 社区已想到但没精力做的具体方向，供学生参考 |
| **提案接收渠道** | ✅ | 社区指定邮箱或其他渠道（**系统外提交**） |
| **Pitch 时段** | ✅ | 社区计划开放的提案沟通时段。格式："2026年X月X日 19:00-21:00" 或 "另行通知" |

## 七、关键注意事项

1. **全年滚动**：所有流程持续开放，不再有固定截止日期
2. **RISC-V 优先**：所有项目首先应支持 RISC-V 架构
3. **导师身份独立**：联络人与导师需用不同邮箱，导师不能同时当学生
4. **禁止内定**：不允许提前承诺中选或要求学生提前开发
5. **信息修改**：需向组委会发邮件申请，建议发布时仔细核对
